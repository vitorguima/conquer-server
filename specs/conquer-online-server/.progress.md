## Learnings (Design Phase)

- Single flat .csproj mirrors Redux layout ŌĆö no multi-project restructuring needed; all new files added as folders within the single project
- `MySqlConnector` preferred over `MySql.Data` (Oracle) ŌĆö async-native, MIT license, avoids LGPL complications
- `runtime:8.0` (not `aspnet:8.0`) confirmed correct base image ŌĆö Redux has no ASP.NET middleware or Kestrel dependency
- Two `TcpListener` instances (auth :9958, game :5816) run in the same process via `Task.WhenAll` ŌĆö no IPC needed
- TQCipher has two modes: pre-auth (K1 only, identity-ish) and post-auth (K2 active after GenerateKeys called with session token); cipher state lives on `ClientSession`, never shared between connections
- RC5 is used ONLY for MsgAccount password decryption ŌĆö not in the game connection path at all
- `TokenStore` is an in-memory `ConcurrentDictionary<ulong, int>` ŌĆö token is consumed (TryRemove) on MsgConnect; no persistence needed
- MySQL 8 `--default-authentication-plugin=mysql_native_password` must be set as Docker command arg ŌĆö it is NOT the default from MySQL 8.0.4+
- `docker-compose.yml` `depends_on` with `condition: service_healthy` is essential ŌĆö MySQL 8 takes 10ŌĆō20s on first start; server must not attempt DB connection before MySQL is ready
- Latin-1 (ISO-8859-1) encoding for CO packet string fields ŌĆö must use `Encoding.Latin1` not `Encoding.UTF8` when parsing fixed-width name fields
- SHA1 password storage format (hex/base64/raw) is still unresolved ŌĆö must be checked against Redux DDL before implementing `ValidateSha1()`; this is the highest-risk unknown for auth correctness
- DMAP tile passability: `0 = passable`, non-zero = blocked; row-major array at offset 8 in the binary file
- Map files (.dmap) are not redistributable TQ IP ŌĆö operator must supply from their own client install; server mounts them via Docker volume
