## Learnings

- **Decision: Fork COServer Redux (5065), not Comet@5017** — Redux has more complete game logic (combat, items, guilds, NPCs); accepts more modernization work in exchange
- Redux is .NET Framework 4.0 / x86 / NHibernate / ManagedOpenSsl.dll — significant modernization required to reach .NET 8 / Linux / Docker
- Modernization path: async socket rewrite, replace NHibernate→Dapper/EF Core 8, AnyCPU, replace OpenSsl dep with System.Security.Cryptography, add Dockerfile
- Redux monolith has no Auth/Game server split — can keep monolithic or refactor; decision for requirements phase
- TinyMap.dll (DMAP parser) must be ported to managed .NET 8 code — no native Linux binary allowed
- Redux uses MySQL 5.6 schema; upgrade to MySQL 8 / MariaDB needed for Docker
- CO 5065 client: community-distributed; TQ has not pursued takedowns for educational/private use
- Redux license: not explicitly stated; community release, assumed public domain by author intent
- Redux v3.0 is described as "fully functional" for 5065 — auth, items, combat, guilds; monster AI is a known TODO
- Alternative reference: Comet@5017 (https://github.com/conquer-online/comet) is a clean .NET 6 async skeleton — useful as architecture reference even if not forked
- CO dev wiki: https://conquer-online.github.io/wiki/ — 200+ packet types, DAT file formats, game constants

## Learnings (Design Phase)

- Single flat .csproj mirrors Redux layout — no multi-project restructuring needed; all new files added as folders within the single project
- `MySqlConnector` preferred over `MySql.Data` (Oracle) — async-native, MIT license, avoids LGPL complications
- `runtime:8.0` (not `aspnet:8.0`) confirmed correct base image — Redux has no ASP.NET middleware or Kestrel dependency
- Two `TcpListener` instances (auth :9958, game :5816) run in the same process via `Task.WhenAll` — no IPC needed
- TQCipher has two modes: pre-auth (K1 only, identity-ish) and post-auth (K2 active after GenerateKeys called with session token); cipher state lives on `ClientSession`, never shared between connections
- RC5 is used ONLY for MsgAccount password decryption — not in the game connection path at all
- `TokenStore` is an in-memory `ConcurrentDictionary<ulong, int>` — token is consumed (TryRemove) on MsgConnect; no persistence needed
- MySQL 8 `--default-authentication-plugin=mysql_native_password` must be set as Docker command arg — it is NOT the default from MySQL 8.0.4+
- `docker-compose.yml` `depends_on` with `condition: service_healthy` is essential — MySQL 8 takes 10–20s on first start; server must not attempt DB connection before MySQL is ready
- Latin-1 (ISO-8859-1) encoding for CO packet string fields — must use `Encoding.Latin1` not `Encoding.UTF8` when parsing fixed-width name fields
- SHA1 password storage format (hex/base64/raw) is still unresolved — must be checked against Redux DDL before implementing `ValidateSha1()`; this is the highest-risk unknown for auth correctness
- DMAP tile passability: `0 = passable`, non-zero = blocked; row-major array at offset 8 in the binary file
- Map files (.dmap) are not redistributable TQ IP — operator must supply from their own client install; server mounts them via Docker volume
