## Learnings

- 5065 client only needs the AUTH/login endpoint repointed. Game IP+port are sent by the auth server at runtime (LoginServer.cs:91-92,108-109 set reply.Info=GAME_IP, reply.ServerPort=GAME_PORT). Patcher must NOT set game IP.
- Three client repoint vectors for 5065: (a) hardcoded auth-host string in Conquer.exe, (b) server.dat binary server-list, (c) `Conquer.exe blacknull` CLI arg. Community-canonical method = runtime DLL injection via ConquerLoader's CLHook.dll (for client version <5717, which is 5065).
- ConquerLoader (github.com/darkfoxdeveloper/ConquerLoader): C# WPF, supports 5065-6736. For <5717 uses CLHook.dll in-memory IP patch, UseDecryptedServerDat=No. **No license declared** -> reimplement technique, do not vendor code.
- server.dat fields (from ServersDatGenerator.cs): ServerName, ServerIP, ServerPort, FlashName, FlashIcon, Child... where ServerIP/ServerPort = LoginHost/LoginPort. For 5717+ it's an XML-template build; for 5065 it's packed/obfuscated and not hand-editable reliably.
- Gotcha: client rejects LAN/internal IPs ("Server.dat is damaged") without an extra exe hex patch -> default to 127.0.0.1 loopback. Must delete client's tqantivirus/anti-cheat folder. AV flags injector DLLs (false positive).
- Binary patch constraints: new IP string <= old length, null-pad, keep terminator, don't extend file. Exact offsets/format are client-build-specific -> MUST inspect operator's real Conquer.exe + server.dat to confirm.
- Discrepancy: LOGIN_PORT defaults to 9959 in Constants.cs:87 but overridden to 9958 via ini (SettingsReader.cs) + README. Patcher default port = 9958; reconcile server-side.
- Repo has NO tests and no lint/CI. New patcher should add an xUnit test project with a synthetic fixture binary for round-trip patch verification. Build only: `dotnet build -c Release`.

## Interview Responses

### Requirements Interview (from requirements.md)
- Mechanism: Static byte-patcher (cross-platform net8.0; rewrite auth string in a COPY of Conquer.exe / server.dat). Injection launcher is OUT of scope for v1.
- UX: CLI only — e.g. `patcher.exe --client <dir> --ip 127.0.0.1 --port 9958`.
- Targeting: Operator-supplied search string (old IP/hostname to find), length-checked + null-padded replacement. No auto-scan in v1.
- Chosen approach: MVP static CLI patcher — set AUTH endpoint only, copy-and-patch with backup + diff report, xUnit fixture tests.

### Design Interview (from design.md)
- Layout: New `src/ClientPatcher/` (net8.0 console) + `src/ClientPatcher.Tests/` (xUnit), matching the repo's multi-project src/ convention; separate from the Linux server solutions.
- CLI parsing: Hand-rolled arg parser, zero new NuGet deps (repo is lean: Dapper/MySqlConnector/Extensions only).
- Report: Human-readable text to stdout (file, offset(s), old→new bytes, counts). No JSON in v1.
- Chosen approach: Isolated new console project + test project; pure byte-rewriter, no server deps.

### Tasks Interview (from tasks.md)
- Granularity: fine (40-60+ atomic tasks, one commit each, [VERIFY] every 2-3).
- Testing: Standard unit + synthetic fixtures (xUnit: PatchEngine, Validator, Parser, EndpointBuilder) + one in-memory integration test, per design test strategy.
- E2E verification: YES (in-memory) — VE runs full pipeline against a temp dir of synthetic fixture files (no real Conquer.exe), asserts backup/length/report. Real-client E2E stays operator-verified, out of CI.
- Chosen approach: Thorough — POC-first phases with full fixture-based coverage and quality gates; in-memory E2E in CI.
