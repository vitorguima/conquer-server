## Learnings (research phase — enter-world)

- **DataAction enum (src/Redux/Enum/DataAction.cs) — ushort numeric values:** SetLocation=74, Hotkeys=75, ConfirmFriends=76, ConfirmProficiencies=77, ConfirmSkills=78, ChangeAction=81, CompleteLogin=130, Jump=133, Teleport=86, GetSurroundings=114, InvisibleEntity=102, NewCoordinates=108. The client's first 1010 is almost certainly **SetLocation(74)**.
- **GeneralActionPacket(1010) wire layout (src/Redux/Packets/Game/[1010] GeneralData.cs):** body 28 bytes (+8 seal). Timestamp@4(u32), UID@8(u32), Data1@12(u32), Data2@16(u32), Data3@20(u16), **Action@22(u16)**. NOTE: the carried hexdump `1800 F203 <ts:4> 01000000 00000000` is the FIRST 12 bytes after the seal-stripped header (len=0x18=24 is the on-wire length field; actual body 28). The Action field sits at offset 22 — re-capture the full 36-byte frame live to read the real subtype (the `01000000` at body+8 is Data1=1, not the action).
- **SetLocation server reply (GameServer.cs:429-437):** echoes the SAME 1010 packet back with Data1=MapID, Data2Low=X, Data2High=Y, THEN sends MapStatusPacket(1110). This is the "jump to my map+coords" reply that clears the loading screen. No map files needed.
- **Player's OWN spawn is NOT auto-sent.** Populate()/MapManager.AddPlayer (src/Redux/Managers/MapManager.cs:32) only insert into map + UpdateSurroundings (spawns OTHER entities). The client renders its own body from 1006 + the SetLocation position. The own-spawn SpawnEntityPacket(1014) is sent ONLY to other players (SpawnByUID) OR back to self on **DataAction.InvisibleEntity(102)** (GameServer.cs:603-609: `user.Send(SpawnEntityPacket.Create(user))`). So minimal stand-still spawn likely needs ONLY: SetLocation echo + MapStatus(1110). 1014-to-self is a fallback if the body doesn't render.
- **SpawnEntityPacket(1014) layout** (src/Redux/Packets/Game/[1014] SpawnEntity.cs): 100+names bytes. UID@4, Lookface@8, X@52, Y@54, Hair@56, Direction@58, Action@59(byte), Level@50/@62, name via NetStringPacker@90.
- **MapStatusPacket(1110):** 16+8 bytes. UID@4, ID@8 (map id), Type@12. In net8 there is no DbMap loaded; can send a minimal one (UID=mapid, ID=mapid, Type=0).
- **Map files NOT required (net8/modernize-m1):** `src/Redux/TinyMapStub.cs` `TinyMapService.Valid(...)` is a STUB that always returns `true`. Original `ChangeMap`/`HandleJump` gate on `Common.MapService.Valid(...)` but with the stub everything passes. Client loads its own .cqmap. Server only needs to send map id + X/Y.
- **Canonical spawnable coords confirmed:** original `CreateDbCharacter` (src/Redux/Objects/Player.cs:688-690) = Map **1002 / X 438 / Y 381** (Twin City). Nov_16_Backup.sql confirms nearly all chars live on map 1002 (~400-480 / ~350-395). ## Learnings (design phase — enter-world)

- **Offset math CONFIRMED against source:** `[1010] GeneralData.cs:94` reads `Action = *(DataAction*)(ptr+22)` (body offset 22). `GameConnection.HandleFrames` (src/Redux/GameConnection.cs:128-130) strips the 2-byte length prefix → `payload = frame[2..]`. So Action is at **payload offset 20** (22−2). The frame keeps its 8-byte seal in payload (payloadLen = frameLen−2 = body+8−2 = body+6), so a 1010 dispatch payload is ~34 bytes — ample for the u16@20 read.
- **SetLocation reply values CONFIRMED (GameServer.cs:429-435):** `packet.Data1 = MapID; Data2Low = X; Data2High = Y; client.Send(packet); client.Send(MapStatusPacket.Create(map))`. So Data1=MapID (NOT Data2 — the task's "Data1=MapID, Data2=(Y<<16)|X" is correct: Data1 holds map, Data2 packs X(low)/Y(high)). Echo reuses Action=74, UID=charUID.
- **Builder convention is body-only + AppendHeader(span, bodyLen+8, type):** existing MsgTalk/HeroInformation write length field = `(bodyLen+8)-8 = bodyLen`; SendGame appends the 8-byte "TQServer" seal. New net8 builders MUST follow this (return body only) and use Span/BinaryPrimitives (NO unsafe, unlike original structs).
- **DbCharacter lives in src/Database/CharacterRepository.cs** (NOT a separate Domain/DbCharacter.cs — that path does not exist). Has MapID(1010)/X(61)/Y(109)/Mesh/Avatar/Level/HealthPoints/Name — all needed fields present. Network.csproj must reference Database.csproj for the new ClientSession.Character field.
- **No Conquer.Packets test project exists yet** — only Crypto.Tests + ClientPatcher.Tests. Design adds a NEW `src/Packets.Tests` xUnit project (mirror Crypto.Tests.csproj) referencing Packets.csproj, added to Conquer.sln, to host the 3 builder-layout assertions + the Action-parse@20 test.
- **Handler boundary decision:** 1010 handling goes in a NEW `ActionHandler` (src/Packets/ActionHandler.cs), a sibling of AuthHandler/GameHandler owned by PacketRouter. GameHandler stays the 1052-only handler. Avoids bloating GameHandler; matches existing router pattern.
- **1014/1110 builders use minimal/zero fields:** net8 has no DbMap (MapStatus → UID=ID=MapID, Type=0) and no equipment/guild/status (SpawnEntity self → those offsets zero, Action=0 stand). 1014 + GetSurroundings(114) stay GATED off by default (A2/FR-7/FR-8 fallbacks).
